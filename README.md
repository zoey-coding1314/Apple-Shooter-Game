# Apple-Shooter-Game
Hit or Miss

A 2D archery game developed in C++ using SplashKit.

✨ Features

• Shoot apples with a bow
• Score tracking
• Collision detection
• Multiple game states
• Sound effects
• Interactive UI

🛠 Technologies

• C++
• SplashKit
• Object-Oriented Programming

🎥 Demo

https://youtu.be/SJ_gl4-AQdM

Build and Run Instructions

Required resources: 
SplashKit Library - Required for graphics, sound, input handling, and game utilities
C++ compiler
Setup steps:
1. Download and install Splashkit Library.
2. Download all the .cpp and .h files to the same workplace.
3. Create a Resources folder in the same workspace as the .cpp files.
4. Download all the required resources: fonts, images, and sounds and add them to the Resources folder.
5. Running the program:
    Compile the project with command:
    clang++ global.cpp hom_draw.cpp hom_update.cpp hit_or_miss.cpp Target.cpp hit_or_miss_main.cpp -l splashkit -o run
6. Run the code:
    ./run
7. Enjoy the game


Longer Description/Project Overview:

Hit or Miss is an extension of the H2 project that I originally developed as my custom program. The core logic of the game remains the same — aiming and shooting arrows, but it evolves through more dynamic rules and states. 
To increase depth, it is planned to be equipped with an improved structure using classes and functions, adaptive mechanics that analyse the player's performance, streak-based mechanics with a mini game and more. 
File input/output will be used to store player scores and progress, allowing the game to store necessary information. The project may be extended into a two-player mode, which encourages competitive and engaging gameplay. 

How You Developed This Project Pitch:

The idea of this project was developed through several factors. Initially, I was quite clueless about what type of project to do because there were no limitations for this project, and I had difficulty generating ideas independently. Thus, I sought some advice from my tutor, who encouraged me to consider developing a game, as game-based projects are interesting and naturally allow for complexity and creativity. 
Besides, I was advised to start a project that I’m interested in. Therefore, this led me to reflect on the games I played in the past. I was inspired by a simple arcade I used to play when I was little: BoxHead. It is a game of fighting armies of zombies with weapons, basically shooting the zombies and staying alive. While the mechanics of that game were far beyond my capability, it inspired me to design a simpler and more achievable project. Which I decided to do something beneath my skill level and retain the core idea of aiming and shooting — a basic archery game that focuses on precision and decision-making.
With this idea in mind, I am able to come up with a lot of following ideas about the game rules, scoring system and how to make it different from other archery games I played before while remaining within the scope of my skills.
Before starting the project, I could already imagine the challenges and problems I would face during the process, such as graphical design and game mechanics. However, I believe that these challenges are manageable with enough determination and hard work. 
In summary, the project was designed through a combination of my personal interests, inspiration from games I played, and constructive advice gained from discussions with my tutor. I hope that this is ambitious enough to meet the HD standard while remaining realistic within my current ability and understanding of the unit content.
